ARCHITECTURE EN COUCHES
======================

┌──────────────────────────────────┐
│    COUCHE PRÉSENTATION (JavaFX)  │
│  ├─ MainController               │
│  ├─ LivreController              │
│  ├─ MembreController             │
│  └─ EmpruntController            │
└──────────────────────────────────┘
                 ↓
┌──────────────────────────────────┐
│    COUCHE SERVICE (Métier)       │
│  ├─ BibliothequeService          │
│  └─ EmpruntService               │
│     (Validations, Règles)        │
└──────────────────────────────────┘
                 ↓
┌──────────────────────────────────┐
│    COUCHE DAO (Données)          │
│  ├─ LivreDAOImpl                  │
│  ├─ MembreDAOImpl                 │
│  └─ EmpruntDAOImpl                │
│     (CRUD avec PreparedStatement) │
└──────────────────────────────────┘
                 ↓
┌──────────────────────────────────┐
│    BASE DE DONNÉES MySQL         │
│  ├─ livres                       │
│  ├─ membres                      │
│  └─ emprunts                     │
└──────────────────────────────────┘


FLUX DE DONNÉES - Exemple : Emprunter un Livre
==============================================

1. LivreController.handleEmprunter()
   │
   └─→ 2. EmpruntService.emprunterLivre(isbn, membreId)
        │
        ├─→ Validation (membre actif, livre disponible, limite)
        │
        ├─→ 3. MembreDAO.findByIntId(membreId)
        │
        ├─→ 4. LivreDAO.findByISBN(isbn)
        │
        ├─→ 5. EmpruntDAO.save(emprunt)
        │
        └─→ 6. LivreDAO.update(livre) [marquer indisponible]
        
   └─→ 7. Afficher alerte succès au contrôleur

POINTS CLÉS D'IMPLÉMENTATION
============================

1. ✅ PreparedStatement uniquement (sécurité SQL injection)
2. ✅ Singleton avec Double-Checked Locking (thread-safe)
3. ✅ Exceptions personnalisées pour la logique métier
4. ✅ Validation stricte des données avant insertion
5. ✅ Encapsulation complète (private + getters/setters)
6. ✅ JavaDoc sur toutes les méthodes publiques
7. ✅ Gestion des ressources avec try-with-resources
8. ✅ Architecture en couches stricte (Controller ≠ DAO directement)
9. ✅ Héritage et polymorphisme (Document, Personne, Empruntable)
10. ✅ Injection de dépendances dans les services
